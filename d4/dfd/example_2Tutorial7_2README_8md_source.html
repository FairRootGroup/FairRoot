4 ## Quick start
5 
6 ### Generate input file
7 Start the script startTuto7GenerateData.sh (in FairRoot/build/bin/) to generate and store random data.
8 A file Tuto7InputFile.root will be produce in the directory FairRoot/example/Tutorial7/macro/datasource.
9 
10 ### Start the process chain
11 Start the script startTuto7All.sh to use the Tuto7InputFile.root as data source.
12 The implemented serialization formats are boost, binary, Root (TMessage).
13 To enable a specific format use e.g.
14 
15 ```bash
16 ./startTuto7All.sh bin
17 ./startTuto7All.sh boost
18 ./startTuto7All.sh root
19 ```
20 
21 Alternatively you can start the script startTuto7AllTuto3.sh to use the testdigi.root from tutorial3 as data source.
22 See tutorial 3 for more details.

24 ## Introduction
25 The Tutorial 7 show how to use three generic devices, namely a generic sampler, processor and filesink.
26 These three devices are policy based designed classes. These devices all inherit from FairMQDevice and from specific template parameters, namely :
27 * The sampler inherit from
28  * a sampler policy (e.g a file reader)
29  * an output policy (serialize)
30 * The processor inherit from
31  * an input policy (deserialize)
32  * an output policy (serialize)
33  * a task policy (process data)
34 * The filesink inherit from
35  * an input policy (deserialize)
36  * a storage policy.
37 
38 Some policy examples (c.f. e.g. BoostSerializer.h) are available in Base/MQ.
